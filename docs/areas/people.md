# People

Goal: make AI readiness visible per person and role without turning enablement into generic training.

## Role Map

- [ ] List roles and core workflows.
- [ ] Identify where AI changes expected output quality or speed.
- [ ] Define required AI fluency per role.
- [ ] Define role-specific risks and boundaries.
- [ ] Assign manager or functional owner for readiness.

## Skill Ladder

- [ ] Level 1: uses approved AI tools safely for low-risk work.
- [ ] Level 2: writes useful prompts and validates output against sources.
- [ ] Level 3: designs repeatable AI-assisted workflows.
- [ ] Level 4: evaluates AI quality, risk, and impact.
- [ ] Level 5: builds or owns AI-enabled systems for a team.

## Enablement

- [ ] Create role-specific examples.
- [ ] Create checklist for safe use: data, source verification, review, and escalation.
- [ ] Run hands-on sessions with real team workflows.
- [ ] Create office-hours path for stuck users.
- [ ] Share working patterns in a central place.

## Verify

- [ ] Ask each person to submit one useful AI-assisted workflow.
- [ ] Review artifacts for accuracy, source use, and safe data handling.
- [ ] Track adoption in real work, not training attendance only.
- [ ] Attach proof artifact: workflow note, reviewed prompt, output example, or manager sign-off.

## Operate

- [ ] Refresh examples when tools or policies change.
- [ ] Pair advanced users with adjacent teams.
- [ ] Track risks, incidents, and useful patterns.
- [ ] Keep readiness visible in team planning.
