# Constitution

## Goal

Represent a checklist to become an AI-ready repository.

## Concept Naming

- Concept names should be one word whenever possible.
- Concept names may use two words when one word would be vague.
- Concept names must be explicit enough to be understandable in the checklist table without opening the detail file.
- Concept filenames must match concept names exactly, using uppercase words and underscores, e.g. `CHECK.md` or `VERIFICATION_MATRIX.md`.
- Prefer concrete repository-readiness language over abstract process language.
