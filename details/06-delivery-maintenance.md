# Delivery And Maintenance Details

## GENERATED_ARTIFACTS

Where: `AGENTS.md`, `docs/generated.md`.

Acceptance:

- Generated paths are listed.
- Source inputs are listed.
- Regeneration command is exact.
- Hand-editing is forbidden.

Example:

```md
- `src/generated/` is generated by `pnpm codegen`; do not edit by hand.
- Source inputs live in `schema/*.graphql`.
- After changing schema files, run `pnpm codegen && pnpm check`.
```

## RELEASE_DELIVERY

Where: `docs/release.md`, release workflow.

Acceptance:

- Version bump locations.
- Changelog source.
- Build/publish/deploy commands.
- Artifact or registry verification.
- Deployed health smoke.
- Failure handling.

Example:

```md
- Run `pnpm check`.
- Publish package: `pnpm publish --access public`.
- Verify: `npm view <pkg>@<version>`.
- Deploy: `pnpm deploy`.
- Smoke: `curl -fsS https://api.example.com/healthz`.
```

## CHANGELOG_POLICY

Where: `CHANGELOG.md`, `AGENTS.md`.

Acceptance:

- User-visible changes require changelog entry.
- Internal-only/test-only changes may skip with reason.
- Format is documented.

Example:

```md
Update `CHANGELOG.md` for user-visible behavior, config, CLI flags, API shape, migrations, or release changes.
```

## SECOND_OPINION

Where: `AGENTS.md`, optional `docs/agents.md`.

Acceptance:

- Trigger conditions.
- Dry-run/preview command.
- Approval boundary for paid/live calls.
- How to cite result.

Example:

```md
Use a second model/tool for stuck bugs, high-risk architecture changes, security-sensitive changes, or large refactors. Preview context first and cite output path/session id in handoff.
```
