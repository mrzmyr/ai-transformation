# Workflow Automation

Goal: automate non-tech business processes and internal work without hiding ownership or decision points.

## Inventory

- [ ] List recurring internal processes by team.
- [ ] Capture trigger, input, output, owner, frequency, tool, and current pain.
- [ ] Mark process as manual, assisted, automated, or agentic.
- [ ] Identify systems of record and approval points.
- [ ] Flag processes with compliance, privacy, legal, or customer-impact risk.

## Prioritize

- [ ] Score by hours saved, cycle time reduction, quality risk, implementation effort, and reversibility.
- [ ] Pick one narrow process per team for first automation.
- [ ] Define before/after metric.
- [ ] Define stop condition where automation must hand back to a human.

## Implement

- [ ] Document workflow steps in plain language.
- [ ] Define tool/API access and least-privilege credentials.
- [ ] Add audit log for automated actions.
- [ ] Add dry-run mode before live execution.
- [ ] Add alert when automation fails or produces low-confidence output.

## Verify

- [ ] Run against historical examples.
- [ ] Compare automated output with human output.
- [ ] Test missing input, conflicting input, duplicate input, and permission failure.
- [ ] Attach proof artifact: before/after timing, dry-run logs, screenshots, or acceptance notes.

## Operate

- [ ] Assign business owner and technical owner.
- [ ] Review automation quality at fixed cadence.
- [ ] Track savings, exceptions, and manual overrides.
- [ ] Retire automation when process or source system changes.
